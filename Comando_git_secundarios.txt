git diff + nome do arquivo especifico ou . para todos os arquivos na pasta -> é o comando que mostra as diferenças entre versões código.
Ele compara o estado atual dos seus arquivos com o último commit (ou entre branches), exibindo exatamente quais linhas foram adicionadas (em verde/+) ou removidas (em vermelho/-). É, essencialmente, o "antes e depois" das suas alterações.

git reset + nome do arquivo -> serve para "tirar" um arquivo do Stage (desfazer o git add) sem apagar o que você escreveu.

git checkout -b <nome-da-branch> é um "atalho" que faz duas coisas ao mesmo tempo:
-b (Create): Cria uma nova branch a partir de onde você está.
checkout (Switch): Muda você imediatamente para essa nova branch.
Sem o -b, o git checkout serve apenas para trocar entre branches que já existem.

git branch -> é o camnado que sinaliza todas as branch do projeto.

git merge <nome da branch> -> é o camando para trazer para branch principal todas alterações e criações feita e determinada branch
